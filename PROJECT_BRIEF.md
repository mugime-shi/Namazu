# PROJECT_BRIEF.md
# Namazu — スウェーデン向け電力価格最適化ダッシュボード

## 1. プロジェクト概要

### 何を作るか
スウェーデン（SE3エリア：ヨーテボリ含む）に住む誰もが使える電力消費最適化ダッシュボード。2025年に導入された15分単位のダイナミック価格（Kvartspris）を活用し、「いつ電気を使えば一番安いか」をリアルタイムで可視化する。

その上に、太陽光発電オーナー向けの「売電 vs 蓄電 vs 自家消費」の最適化シミュレーション機能を載せる二層構造。

### 二層構造
- **Layer 1（消費最適化）**：ソーラーパネルもEVも持っていないフラット住まいの自分自身が毎日使えるツール。「今日の一番安い時間帯はいつか」「洗濯機・食洗機をいつ回すと月いくら節約できるか」
- **Layer 2（売電最適化）**：太陽光発電オーナー向け。Nord Poolスポット価格に連動した「今売るか、蓄電して後で売るか、自家消費するか」の最適化シミュレーター

### ひとことで言うと
「ヨーテボリのフラットに住むエンジニアが、自分の電気代を最適化するために作ったツール。その裏側には、日本で7年間FIT売電バックエンドを設計・運用してきた経験がある」というポートフォリオプロジェクト。

### なぜ二層構造なのか
- 自分自身がLayer 1のユーザーだから、「作って終わり」にならない
- Layer 1だけでも汎用ツールとして完成している → フラット住まいの大多数に価値がある
- Layer 2はLayer 1の延長で、プロシューマー向けの付加価値 → ドメイン知識のアピール
- 面接で「自分で毎日使っています」と言えるのは、「就活用に作りました」より圧倒的に強い

## 2. なぜ作るのか — ビジネス上の背景

### 日本側の文脈
- 2019年11月以降、家庭用太陽光発電のFIT（固定価格買取制度）が順次満了（卒FIT）
- FIT期間中は40円/kWh超で売電できたが、卒FIT後は大手電力会社に7〜8円/kWhで売電
- 「売るより自家消費した方が得」→ 蓄電池需要が急増
- TRENDEは卒FIT家庭向けにP2P電力取引で付加価値を提供する戦略
- 伊藤忠の子会社としてSmart Star蓄電池 + P2P取引プラットフォームを展開中

### スウェーデン側の文脈（消費者として）
- 2025年からEU基準の15分単位ダイナミック価格（Kvartspris）が導入
- SE3エリア（ヨーテボリ）では深夜と夕方でスポット価格が7〜8倍違うことがある
- Tibber、Greenely がアプリで価格可視化を提供しているが、まだ発展途上
- スウェーデンの人口の約42%が戸建てだが、都市部の大多数はフラット住まい
- フラット住まいでもダイナミック価格の恩恵を受けられるのに、ツールが不足している
- **自分自身がMajornaのフラットに住んでいて、まさにこのツールのユーザー**

### スウェーデン側の文脈（プロシューマーとして）
- 2026年1月からマイクロプロデューサー向け税控除（60öre/kWh）が完全廃止
- 売電で得られるのはNord Poolスポット価格のみ（平均30〜50öre/kWh）
- 購入価格（200öre/kWh以上）との差が大きく、「売るより自家消費した方が得」
- 蓄電池設置が前年比34%増（2025年に75,000人が税控除を利用）
- TibberやGreenely がVPP（仮想発電所）やスマート充電で急成長中

### 構造の同型性 — これがプロジェクトの核心
日本とスウェーデンで「固定インセンティブの終了 → 動的市場への移行」という同じ構造的変化が起きている。異なるのは：
- 日本：FIT固定価格 → 卒FIT変動価格（＋P2P取引の可能性）
- スウェーデン：税控除固定額 → スポット価格のみ（＋VPP・フレキシビリティ市場）

そしてこの変化は、プロシューマーだけでなく一般消費者にも影響する。Kvartsprisの導入で、スウェーデンの全電力消費者がリアルタイムの価格変動にさらされるようになった。**Layer 1はこの消費者全体の問題を、Layer 2はプロシューマー固有の問題を解く**。

**両方の市場を知っている人間にしか語れないストーリー**がここにある。

## 3. 競合分析 — 何がすでにあって、何がないか

### Layer 1（消費最適化）の既存プレイヤー
| サービス | 種別 | 特徴 | 制約 |
|---|---|---|---|
| **Tibber App** | 商用（月49 SEK） | 15分価格表示、消費追跡、スマート充電、通知 | Tibber契約者限定 |
| **Greenely App** | 商用（月69 SEK） | 同様の機能、一部無料利用可 | フル機能は契約者限定 |
| **statsskuld.se/elpris** | 無料サイト | 時間帯別価格チャート、家電コスト計算 | 表示のみ、パーソナライズなし |
| **WattSmart** | 無料サイト | 今日・明日の最安/最高時間帯 | 表示のみ |
| **Elpriskollen.se** | 公的サイト（Ei運営） | 電力契約タイプの比較 | 契約比較のみ、価格最適化なし |
| **Home Assistant + Nord Pool** | OSS | 高度にカスタマイズ可能 | 設定が技術者向け、一般消費者には難しい |

### Layer 2（売電最適化）の既存プレイヤー
| サービス | 特徴 | 制約 |
|---|---|---|
| **Tibber Grid Rewards** | 蓄電池VPP最適化 | Homevoltバッテリー限定 |
| **Greenely** | ソーラー売電スポット価格連動、+5öre/kWh上乗せ | 契約者限定 |
| **CheckWatt** | バッテリー充放電最適化 | プロ/ハードウェア連携前提 |

### Namazuの差別化ポイント
1. **電力会社に依存しない独立ツール**。契約前の比較検討にも使える
2. **パーソナライズされたシミュレーション**。消費量入力で「固定 vs ダイナミック」の月額比較が可能（無料サイトにはない）
3. **税控除廃止（2026年1月）に特化したソーラー売電シミュレーション**。これは確認した限りどこもやっていない
4. **ポートフォリオとしての価値**。競合を分析した上で作っている事実自体が、プロダクト思考の証明になる

## 5. 自分（Shin）の強みの正直な整理

### 直接的な強み（7年の実務）
- 太陽光発電の買取・売電システムのバックエンド全般
  - 電力データの取得・保存（スマートメーター連携）
  - FIT買取価格のスクレイピング取得
  - 売電収益の価格計算・請求処理
  - ユーザー向けマイページ（消費・収益の可視化）
- インフラ運用（AWS）
- Ruby on Rails による本番システムの設計・維持保守

### 知識はあるが手を動かしていない領域
- P2P電力取引システム自体（設計・実装は別の人、概要は理解）
- 機械学習による需給予測（概念は理解、実装経験はこれから）

### 持っていないもの（正直に）
- EV車を持っていない、フラット住まいで太陽光パネルもない
- スウェーデンのエネルギー市場の実務経験
- ヒートポンプやV2Hの実体験

### だからこそこのプロジェクト
- **自分自身がユーザー**。Majornaのフラットに住み、Kvartsprisの影響を受ける消費者として毎日使える
- EVもソーラーも持っていなくても、「バックエンドを7年運用してきた」事実は強い
- Pythonで作ることで、Rubyの本番コードとの距離が明確（知財問題ゼロ）
- 「知らない市場を調べて、自分の経験を転用する」プロセス自体がスキルの証明
- 「就活用に作りました」ではなく「自分の生活のために作って、それが仕事の経験と繋がった」が最強のストーリー

## 7. スコープの線引き

### MVP Phase 1 — 消費最適化（自分が毎日使うもの）
- SE3エリアのNord Poolスポット価格をリアルタイム取得・表示
- 今日・明日の15分単位の価格推移チャート
- 「今安い / 高い / 普通」のシンプルなインジケーター
- 「洗濯機・食洗機をいつ回すと一番安いか」のレコメンド
- 月間の消費コストシミュレーション（固定価格契約 vs ダイナミック価格の比較）

### MVP Phase 2 — 売電最適化（プロシューマー向け拡張）
- 太陽光発電量のシミュレーション（天候データ連動）
- 「売電 vs 蓄電 vs 自家消費」の収益比較シミュレーション
- 月間売電収益ダッシュボード
- 税控除廃止前後の収益比較

### Nice to have（余裕があれば）
- 蓄電池の充放電最適化アルゴリズム（簡易版）
- 複数月の履歴データによるトレンド分析
- 日本のFIT制度との比較ビュー（面接での説明資料として）
- 英語 / 日本語 / スウェーデン語の切り替え

### 明確にやらないこと
- 実際の電力取引・決済
- リアルタイムのスマートメーター連携
- P2P取引機能（これは別プロジェクトの可能性として言及するのみ）
- モバイルアプリ（ただしレスポンシブWebで十分使える設計にする）
